# 운영 매뉴얼

관리자(`/admin`)가 일상적으로 수행하는 작업 안내. 로그인은 `/admin/login` — 계정 발급은 [admin-setup.md](./admin-setup.md) 참고.

## 도서 등록

**대시보드 → 도서 관리 → 새 도서 등록** (`/admin/books/new`).

1. **파일 업로드** — DOCX 파일을 드래그하거나 선택 (최대 20MB, `.docx`만 지원 — 구버전 `.doc`나 암호화된 파일은 거부됨). 선택 즉시 실제 변환 결과가 오른쪽 미리보기 패널에 뜬다(저장 전 미리보기, 저장되는 것과 동일한 변환 로직 사용).
2. **도서 정보** — 제목(필수)·저자·요약·슬러그(URL, 비우면 제목에서 자동 생성)·표지 이미지(PNG/JPEG/WebP, 최대 5MB)·카테고리·태그.
3. **공개 상태 / 공개 범위** 설정 — 아래 "공개 상태" 절 필독. **⚠️ 기본값은 초안(draft)이다 — 등록만 하고 상태를 안 바꾸면 링크를 열어도 아무것도 안 보인다.**
4. **등록** 클릭 → 저장 후 도서 관리 목록으로 이동.

DOCX 작성 방법(제목 스타일, 지원되는 서식 등)은 [docx-authoring-guide.md](./docx-authoring-guide.md) 참고.

## 공개 상태(status) / 공개 범위(visibility)

이 둘은 **서로 다른 축**이고 함께 작동해야 도서가 보인다.

| status | 의미 |
|---|---|
| `draft` (기본값) | 작성/검토 중 — **어떤 링크로도 열람 불가**, 목록에도 안 뜸 |
| `published` | 공개 대상 |
| `archived` | 보관 — published에서 내려간 상태, 마찬가지로 안 보임 |

| visibility | 의미 (status=published일 때만 의미 있음) |
|---|---|
| `private` | 관리자만 열람 가능 |
| `unlisted` | 도서관 목록엔 안 뜨지만, 공유 링크(`/share/[token]`)로는 로그인 없이 열람 가능 |
| `public` | 도서관 목록·검색에 노출, `/books/[slug]`로 누구나 열람 |

**즉 홈페이지·`/books/[slug]`로 보이려면 `status=published` AND `visibility=public`이어야 한다.** 도서 관리 목록에서 상태/공개범위 드롭다운으로 바로 바꿀 수 있다(저장 버튼 없이 즉시 반영).

## 도서 관리 (`/admin/books`)

목록의 "관리" 컬럼 아이콘:

- **수정(연필)** — 제목/카테고리/공개 설정 변경, DOCX 재업로드(비우면 기존 콘텐츠 유지), 표지 교체.
- **재변환(순환 화살표)** — 원본 DOCX를 다시 업로드하지 않고, 이미 저장된 원본으로 변환 파이프라인만 재실행(`book-originals` 버킷의 원본 사용). 스타일 맵이나 sanitize 규칙이 바뀐 뒤 기존 도서들을 일괄 갱신할 때 유용.
- **링크 복사** — `allow_share`가 켜져 있고 비공개가 아닐 때만 활성. `public` 도서는 `/books/[slug]`, `unlisted` 도서는 최초 클릭 시 `share_token`을 발급해 `/share/[token]`을 복사.
- **QR 코드** — 위 링크의 QR 코드 표시.
- **공유 링크 재생성** (unlisted 도서만) — 기존에 공유한 링크를 전부 무효화하고 새 토큰 발급. **되돌릴 수 없음** — 확인창 표시됨.
- **원본 다운로드** — 원본 DOCX 파일 다운로드.
- **삭제** — 확인창 후 영구 삭제.

### 표지 이미지 재업로드가 안 보일 때

표지를 바꿔도 예전 이미지가 계속 보이면 브라우저/CDN 캐시 문제였다 — 현재는 매 업로드마다 URL에 캐시 무효화 파라미터(`?v=`)가 붙도록 고쳐져 있어(`src/lib/upload/cover.ts`) 강력 새로고침 없이도 바로 반영된다. 그래도 안 바뀌면 실제 업로드 자체가 실패한 것이니 저장 시 나오는 경고 메시지를 확인한다.

## 공유 기능

- **공유 허용(allow_share)**: 꺼두면 관리자 목록의 링크복사/QR 버튼과 Reader 안의 공유 버튼이 모두 사라진다. 이미 나눠준 링크 자체는 계속 열람 가능 — 이 스위치는 "공유 도구 노출 여부"만 제어하고 접근 권한 자체는 건드리지 않는다.
- **다운로드 허용(allow_download)**: 공개 Reader에서 원본 파일 다운로드 허용 여부.

## 대시보드 (`/admin`)

- 통계 카드(총 도서 수/공개 도서 수/링크 전용 도서 수/총 조회 수) — 선택한 기간 대비 증감(`+N`) 표시.
- **조회수 통계** 그래프 — 최근 7/30/90일 중 선택.
- **인기 도서** — 선택 기간 내 조회 수 상위 5권.
- **최근 조회** — 최근 조회된 도서 이벤트 목록(상대 시각 표시).
- **최근 등록 도서** — 최근 등록순 5권, 상태/공개범위 배지.

조회 수는 Reader를 열 때(페이지 넘김이 아니라 "열람" 시점) 익명 ID 기준으로 기록되며, 같은 브라우저가 30분 내 재방문하면 중복 집계하지 않는다.

## 카테고리 / 사용자 관리

- **카테고리 관리** (`/admin/categories`): 카테고리 이름·슬러그 CRUD. 카테고리 삭제 시 연결된 도서의 분류만 해제되고 도서 자체는 지워지지 않는다.
- **사용자 관리** (`/admin/users`, `admin` 이상만 접근): 관리자 목록 조회. 신규 관리자 추가는 [admin-setup.md](./admin-setup.md)의 API 절차를 따른다 — 이 화면에는 아직 "초대" 버튼이 없다.

## 로그아웃

사이드바 하단 **로그아웃** — Supabase 세션을 지우고 `/admin/login`으로 이동.
